# 字节系面试题型总结

这份文档基于你收集到的面经，目的是提炼出更高频、更值得优先准备的模式。

重点不是逐条记忆公司和岗位，而是回答三个问题：
- 他们到底爱考什么
- 对你这种背景来说，什么最该优先补
- 每类题该怎么答

---

## 一、整体规律

从这些样本里，最明显的规律是：

### 1. 面试通常不是“只聊项目”

更常见的是：
- 项目细聊
- 一道中低难度算法题
- 一两个基础深度学习 / Transformer 问题
- 一个开放式业务或系统设计问题

也就是说，项目很重要，但不是全部。

### 2. 算法题难度通常不一定高，但覆盖面很散

高频出现的不是特别偏竞赛的题，而是：
- 链表
- 树
- DP
- 二分 / 有序数组
- 概率 / 采样
- 回溯
- 字符串处理

这些题的共同点是：
- 不一定最难
- 但非常考基本功
- 很容易在项目聊完后被用来“补刀”

### 3. Transformer / attention 基础非常高频

从这些面经看，多模态、语音、AIGC、大模型岗位都可能问：
- self attention
- scaled dot-product attention
- transformer 推理复杂度
- MHA / MLA
- CLIP / LoRA / Stable Diffusion

所以你不能只准备项目，不准备基础模型原理。

### 4. 开放题和架构题非常常见

例如：
- 妆容迁移怎么做
- 汽车智能对话机器人怎么做
- 多模态如何做文本到图像生成
- 长 context 怎么支持
- embedding 如何做

这说明面试官很看重：
- 结构化拆解问题的能力
- 算法方案设计能力
- 工程感

## 二、按题型归类后的高频题

### A. 链表 / 树 / 基础数据结构

面经里出现的题：
- 反转链表
- 合并 K 个升序链表
- 链表合并
- 搜索二叉树相关
- 二叉树中序遍历（非递归）

结论：
- 这是最高频题型之一
- 难度通常不夸张，但必须写熟

你必须熟练：
- 反转链表
- 合并两个有序链表
- 合并 K 个升序链表
- 二叉树前中后序遍历
- 非递归遍历
- BST 基础判断和搜索

### B. DP

面经里出现的题：
- 编辑距离
- 最长公共子序列
- 零钱兑换
- 接雨水

结论：
- DP 是稳定高频题型
- `编辑距离 / LCS / 零钱兑换` 必须会

### C. 数组 / 二分 / 有序查找

面经里出现的题：
- 排序数组中查找元素
- 第 k 个缺失数字
- 有序数组求不重复平方和个数
- 使用数组数字组成最大且小于 xxx 的数

结论：
- 二分和数组题必须熟
- 很多题其实是“有序结构 + 规律”

### D. 回溯 / 搜索 / 组合计数

面经里出现的题：
- threshold 子列表计数
- 组成小于某数的最大数
- 一些字符串恢复 / 找规律题

结论：
- 回溯不一定高频，但出现时容易把人搞乱
- 你至少要会“暴力 -> 剪枝 -> 排序”的思路

### E. 概率 / 随机采样

面经里出现的题：
- 概率抽样
- rand7 实现 rand10

结论：
- 经典题不多，但一旦问到，很多人会慌
- 至少要会 `rand7 -> rand10`

### F. 多模态 / Transformer 基础题

面经里出现的题：
- self attention
- scale dot-product attention
- MLA
- 推理复杂度
- CLIP 和 LoRA 细节

结论：
- 这类题和你的岗位强相关
- 不只是“知道概念”，要能写公式或代码骨架

### G. 开放题 / 系统题

面经里出现的题：
- 妆容迁移怎么做
- 汽车智能对话机器人怎么做
- Text2SQL 精度怎么提
- context 超长怎么办
- 多模态怎么做文本到图像生成

结论：
- 很多面试官在开放题里看的是架构能力
- 你要学会“先拆目标，再拆模块，再拆数据/训练/评测”

## 三、对你最重要的优先级

### 第一优先级
- 链表
- 树
- 编辑距离
- LCS
- 零钱兑换
- 岛屿数量
- 排序数组查找
- self attention
- scaled dot-product attention

原因：
- 高频
- 基础
- 最容易失分

### 第二优先级
- rand7 -> rand10
- 第 k 个缺失数字
- 接雨水
- 回溯题
- 搜索二叉树基础题
- 字符串处理题

### 第三优先级
- BWT 逆过程
- 开放题
- ML / DL 数学推导题
- Python 底层概念题

## 四、每类题怎么准备

### 1. 链表 / 树

准备方法：
- 每题自己写 2 遍
- 不看答案能写出指针版本
- 主动讲清楚边界条件

面试要点：
- 先说数据结构关系
- 再说迭代还是递归
- 最后说复杂度

### 2. DP

准备方法：
- 每题都用“状态定义 -> 转移方程 -> 初始化 -> 遍历顺序”讲一遍

面试要点：
- 不要上来就背代码
- 一定要先讲状态

### 3. 数组 / 二分

准备方法：
- 熟悉左右边界模板
- 熟悉“答案具有单调性”的判断

面试要点：
- 先判断能不能二分
- 再说 check 函数是什么

### 4. attention / Transformer

准备方法：
- 记公式
- 记 shape
- 记复杂度
- 至少能手写简化版代码

面试要点：
- 为什么要除 `sqrt(d)`
- mask 放哪
- 输出 shape 是什么

### 5. 开放题

准备方法：
- 用统一模板回答

模板：
1. 先定义问题目标
2. 再拆输入 / 输出
3. 再拆模型或系统模块
4. 再讲数据和训练
5. 最后讲评测和风险

## 五、你现在最值得背的 10 个判断

1. 字节系不会只问项目，常常项目后面接一道基础题。
2. 链表和树非常高频。
3. 编辑距离、LCS、零钱兑换是必刷 DP。
4. attention / self attention 必须能手写。
5. 二分和有序数组题必须熟。
6. 开放题不能乱聊，要结构化回答。
7. 你最容易被追问的是“项目很强，那基本功呢？”
8. 所以 coding 和 Transformer 基础要稳。
9. 面试官不一定出特别难题，但会看你是否扎实。
10. 对你来说，稳住中低难度高频题比刷冷门 hard 更重要。

---

## 六、推荐复习策略

### 第一轮
- 只刷高频题
- 不求多，求写顺

### 第二轮
- 按题型总结模板
- 链表 / DP / attention 各做一套

### 第三轮
- 做口述训练
- 练“先讲思路，再写代码”

---

## 七、最后提醒

从这批面经反推，字节系最常见的失分点不是“不会最难题”，而是：
- 项目很好，但基础题不稳
- 会写代码，但不会讲复杂度
- 会讲业务，但 Transformer 基础不扎实

所以你的复习重心应该是：
- 项目继续保持优势
- 高频题要稳
- attention 要会
- 开放题要能拆
