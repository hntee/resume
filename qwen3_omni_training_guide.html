一、Qwen3 Omni 核心架构

Vision Encoder 继承 Qwen3-VL，提取视频帧视觉特征
Audio Encoder (AuT) 2000 万小时音频预训练，替代 Whisper
Text Embedding 文本 Prompt 编码
↓
TMRoPE 时间对齐 在 RoPE 位置编码上做多模态时间轴扩展，让音频的第 3 秒和视频的第 3 秒对齐到同一个 position space
↓
共享 Attention 融合层 视觉 / 音频 / 文本 token 在统一 attention 里直接交互 — 不是 late fusion，是原生 Early Fusion
↓
Thinker（思考者） MoE 架构 (30B 参数 / 3B 激活) · 统一多模态推理 · 文本 token 生成 · 动态路由到专家（数学 / 逻辑 / 视听）
↓
文本输出 Caption / QA / 推理结果
Talker（表达者） 接 Thinker 隐状态 → 多码本自回归 → 流式语音 (可选关闭)

InternVL vs Qwen3 Omni

维度 InternVL 旧版 Qwen3 Omni 现版
架构本质Vision-Language (纯VL)统一多模态 Omni
视频处理抽帧 → 多图 patch完整视频流
音频处理外部 ASR / 旁路输入主干统一建模
融合方式Late Fusion (Projector)Early Fusion (共享 Attention)
时间对齐无TMRoPE
模型结构ViT → Proj → LLMMoE + Thinker-Talker
输出能力纯文本文本 + 语音
参数规模各版本不同30B (3B 激活)
面试关键表述：InternVL 本质上是 VL 模型，视频靠抽帧理解，音频不在主干里。Qwen Omni 是统一多模态架构，视觉、音频、文本在共享 Attention 层融合，时间轴通过 TMRoPE 对齐。核心区别不只是"多了个音频输入"，而是训练目标和结构都从纯 VL 升级成了统一多模态。

二、四个核心技术点

1. Thinker-Talker

  • Thinker 负责统一理解、推理和文本输出
  • Talker 接 Thinker 隐状态，自回归生成语音 token
  • 文本推理和语音生成解耦，但端到端联动
  • 项目中 ENABLE_AUDIO_OUTPUT=0 关闭 Talker

2. TMRoPE

  • 在 RoPE 上做 time-aligned multimodal 扩展
  • 音频时间轴和视频时间轴对齐到同一 position space
  • 解决"画面 3 秒，语音 3 秒"的跨模态对位
  • 项目中 ADD_TIMESTAMP=true 利用此能力

3. Audio Encoder (AuT)

  • 2000 万小时音频数据预训练
  • 替代 Whisper，直接输出音频特征
  • 不经过 ASR 文字中间表示
  • 以原始特征参与多模态推理

4. 共享 Attention

  • 视觉/音频/文本 token 统一 attention 交互
  • 不是 late fusion，是 Early Fusion
  • 比"跑 ASR 再拼文本"效果更好
  • 这才是"原生 Omni"

三、项目工程改造：Audio-in-Video 显式接入

Step 1
统一入口 process_mm_info()

在 qwen_omni_utils/v2_5/__init__.py 中，将音频处理和视觉处理统一收敛到一个函数：process_mm_info(conversations, use_audio_in_video)

Step 2
从 video 显式抽取 audio

audio_process.py 中，当 use_audio_in_video=True 且输入类型为 video 时，先用 av.open 检查音轨，再用 librosa.load(sr=16000) 读出 16kHz 单声道音频

Step 3
video + audio 一起送进 vLLM

auto_infer_omni.py 中，prepare_inputs_for_vllm() 把 video、audio、image 打包进 multi_modal_data，显式传 use_audio_in_video=True

Step 4
业务价值

模型拿到真实音轨而不是从画面"猜"音频，音画联合 caption、QA revise、音频事件理解都更准确

Loss 设计（三合一）

sigmoid

  • 标准 DPO loss
  • 正样本概率 > 负样本概率

bco_pair

  • Binary Classifier Optimization
  • 二分类辅助信号

sft

  • SFT 正则化
  • 防止偏离基座太远
loss_type = "sigmoid" "bco_pair" "sft" loss_weights = 1.0 1.0 1.0
DPO 数据构造：对每条视频生成多个 caption，高质量为 chosen（人工审核 / 高分），低质量为 rejected（含幻觉 / 时间错误 / 音画不一致）

五、面试口语版

之前我们纯 VL 版本主要是用 InternVL 来训练，本质上是视觉编码器加语言模型，视频更多还是通过抽帧的方式去理解，音频并不是主干统一建模的一部分。后来切到 Qwen Omni 后，结构上最大的变化是它本身就是统一多模态架构，视频、音频和文本可以在同一套理解链路里融合。官方这套里比较关键的是 Thinker-Talker 设计，以及用 TMRoPE 去对齐音频和视频时间轴。

我们本地又做了一层接入改造，在 qwen_omni_utils 里把视频里的音轨显式抽出来，并和视频一起送进推理链路，所以它对有声视频 caption 更自然，也更适合后面的音频理解和 revise。

训练方面，我们先做 SFT 用 Megatron 多机多卡全参数微调，冻住 ViT 和 Aligner 只训 LLM。然后做 DPO 偏好优化，用 sigmoid + bco_pair + sft 三合一 loss。最后用 GRPO 在线强化学习 — 奖励函数是 QA accuracy 加 Timestamp IoU 双信号，用外部 Qwen3-32B 当 judge。最终累计产出数亿级 caption 数据，各项指标准确率 80% 以上。
